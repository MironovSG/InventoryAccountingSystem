# Полное описание функционала системы учета ТМЦ

## Обзор системы

Веб-приложение для автоматизации процессов учета, хранения, выдачи и контроля расхода товарно-материальных ценностей в сервисном центре производственного предприятия.

## Схема работы с заявками

### Полный жизненный цикл заявки:

```
1. СПЕЦИАЛИСТ создает заявку
   └─> Статус: ACCEPTED (Принята)
   └─> МОЛ получает push-уведомление: "Новая заявка №XXX"
       │
       ↓
2. МОЛ выбирает заявки и нажимает "Взять в работу"
   └─> Статус: IN_PROGRESS (В работе)
   └─> СПЕЦИАЛИСТ получает push: "Заявка №XXX принята в работу"
       │
       ↓
3. МОЛ обрабатывает заявку
   └─> Статус: AWAITING_ISSUANCE (Ожидает выдачи)
       │
       ↓
4. МОЛ выдает материалы и нажимает "Выдать"
   └─> Статус: ISSUED (Выдана)
   └─> СПЕЦИАЛИСТ получает push: "Материалы выданы"
   └─> Остатки материалов автоматически обновляются
   └─> Создается запись о движении ТМЦ
       │
       ↓
5. Заявка автоматически сохраняется в ИСТОРИИ
   └─> Доступна для просмотра всем пользователям
   └─> Доступна для экспорта в отчеты
```

### Альтернативный сценарий (отклонение):

```
МОЛ отклоняет заявку
   └─> Статус: REJECTED (Отклонена)
   └─> СПЕЦИАЛИСТ получает push: "Заявка №XXX отклонена. Причина: ..."
   └─> Заявка сохраняется в истории с причиной отклонения
```

## Технические особенности

### Push-уведомления

**Реализация:**
- Таблица `notifications` в базе данных
- API endpoints для работы с уведомлениями
- Автоматическая отправка при изменении статусов
- Polling каждые 30 секунд для обновления

**Триггеры отправки:**
1. Создание новой заявки → МОЛ
2. Принятие заявки в работу → Специалист
3. Выдача материалов → Специалист
4. Отклонение заявки → Специалист

### Три склада

**Типы складов:**
- **CONSUMABLES** - Расходные материалы
- **SPARE_PARTS** - Запасные части
- **COMPONENTS** - Комплектующие

Каждый материал привязан к конкретному складу через поле `warehouse_type`.

### Рабочие группы

**Три группы специалистов:**
- **АРМ** - Автоматизированное рабочее место
- **ГОТО** - Группа оперативно-технического обслуживания
- **ГОКС** - Группа оперативно-контрольных систем

Отображается в заявках и истории.

### Экспорт данных

**Форматы:**
- 📄 **PDF** - для печати и архивирования
- 📊 **Excel** - для дальнейшей обработки

**Что можно экспортировать:**
- Заявки (выбранные через checkbox)
- История заявок (выбранные записи)
- Отчеты по расходу материалов
- Аналитические отчеты

### Автоматические процессы

✅ **Автосохранение в истории** - все заявки автоматически попадают в историю
✅ **Обновление остатков** - при выдаче материалов остатки обновляются автоматически
✅ **Отправка уведомлений** - автоматическая при изменении статусов
✅ **Генерация номеров заявок** - автоматическая по формату REQ-YYYYMMDD-XXXX
✅ **Цветовая индикация** - автоматический расчет уровня запасов

## API Endpoints

### Для уведомлений:
```
GET  /api/notifications               - все уведомления
GET  /api/notifications/unread        - непрочитанные
GET  /api/notifications/unread/count  - количество непрочитанных
PUT  /api/notifications/{id}/read     - отметить прочитанным
PUT  /api/notifications/read-all      - отметить все
```

### Для заявок:
```
GET  /api/requests/my-requests        - мои заявки
POST /api/requests                     - создать заявку
GET  /api/requests/{id}               - детали заявки
PATCH /api/requests/{id}/status       - изменить статус
POST /api/requests/{id}/approve       - одобрить
POST /api/requests/{id}/issue         - выдать материалы
POST /api/requests/{id}/reject        - отклонить
```

### Для материалов:
```
GET  /api/materials/active            - активные материалы
GET  /api/materials/search?query=...  - поиск материалов
GET  /api/materials/{id}              - материал по ID
POST /api/materials                    - создать (только МОЛ)
PUT  /api/materials/{id}              - обновить (только МОЛ)
DELETE /api/materials/{id}            - удалить (только МОЛ)
```

### Для экспорта:
```
GET /api/reports/requests/export/excel?startDate=...&endDate=...
GET /api/reports/requests/export/pdf?startDate=...&endDate=...
```
