# Инструкция по запуску юнит-тестов

Тесты расположены в каталоге **`src/test/java/com/systemtmc/inventory/`** и покрывают семь категорий: время отклика, доступность, надёжность, безопасность, адаптация интерфейса, резервное копирование и защита от внешних воздействий.

---

## Запуск из IntelliJ IDEA

### 1. Запуск всех тестов проекта

- В **Project** откройте папку **`src/test/java`** → **`com.systemtmc.inventory`**.
- Правый клик по пакету **`com.systemtmc.inventory`**.
- Выберите **Run 'Tests in com.systemtmc.inventory'**.

Либо через меню: **Run** → **Run...** → выберите конфигурацию тестов (если уже создана).

### 2. Запуск одного тестового класса

- Откройте нужный класс, например **`SecurityTest.java`**.
- Правый клик по имени класса или по иконке класса в левой колонке.
- Выберите **Run 'SecurityTest'**.

Либо нажмите зелёную стрелку слева от объявления класса и выберите **Run**.

### 3. Запуск одного тестового метода

- Откройте класс с тестами.
- Слева от имени метода (например, `protectedEndpointsReturn401WithoutToken`) нажмите зелёную стрелку.
- Выберите **Run 'protectedEndpointsReturn401WithoutToken()'**.

### 4. Просмотр результатов

- После запуска внизу откроется панель **Run**.
- Вкладка с названием теста (например, **SecurityTest**) показывает:
  - зелёные галочки — тесты прошли;
  - красные крестики — тесты упали (подробности и стек — в том же окне).
- В консоли (под списком тестов) выводятся сообщения вида  
  `[Security] ...`,  
  `[Availability] ...` и т.д. — их можно использовать для демонстрации результата.

---

## Запуск из командной строки (Maven)

В корне проекта выполните:

```bash
mvn test
```

Запуск только тестов указанного класса:

```bash
mvn test -Dtest=SecurityTest
```

Запуск тестов по шаблону имени:

```bash
mvn test -Dtest=*Test
```

Результаты появятся в консоли; отчёт также можно посмотреть в **`target/surefire-reports/`**.

---

## Описание тестовых классов и категорий

| Класс | Категория | Что проверяет |
|-------|-----------|----------------|
| **ResponseTimeTest** | Время отклика | Время ответа на запросы не превышает заданный порог (3 сек). |
| **AvailabilityTest** | Доступность системы | Критические эндпоинты доступны; вход и защищённые пути отвечают ожидаемым статусом. |
| **SecurityTest** | Безопасность | 401 без токена, 400 при неверных данных, валидация входа, кодирование пароля (BCrypt). |
| **ReliabilityTest** | Надёжность | Повторный вход возвращает токен; корректный JSON не даёт 500. |
| **InterfaceAdaptationTest** | Адаптация интерфейса | Ответ API в формате JSON и наличие обязательных полей контракта (token, userId, username и т.д.). |
| **BackupTest** | Резервное копирование | Возможность чтения данных через репозиторий (предпосылка для бэкапа). |
| **ExternalProtectionTest** | Защита от внешних воздействий | Пустой/некорректный JSON и очень длинный ввод не приводят к 500. |

## Требования

- **JDK 17**
- **Maven** (для запуска через `mvn test`)
- **Профиль Spring:** тесты выполняются с профилем **`test`** (используется `application-test.yml`: in-memory H2, отключён Flyway, тестовый JWT).

Перед первым запуском в IDEA убедитесь, что проект собран: **Build** → **Build Project** (или **Ctrl+F9**).
