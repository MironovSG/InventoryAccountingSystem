## ➕ Добавление нового пользователя

### Шаг 1: Открытие формы

Нажмите кнопку **"➕ Добавить пользователя"**

### Шаг 2: Заполнение формы

Откроется модальное окно с полями:

#### Обязательные поля (отмечены *)

1. **Логин*** - Уникальное имя для входа
   - Пример: `petrov_aa`
   - Только латинские буквы, цифры, подчеркивание

2. **Пароль*** - Пароль для входа
   - Минимум 6 символов
   - Рекомендуется использовать цифры и буквы

3. **Email*** - Электронная почта
   - Пример: `petrov@company.com`
   - Должен быть уникальным

4. **Имя*** - Имя пользователя
   - Пример: `Александр`

5. **Фамилия*** - Фамилия пользователя
   - Пример: `Петров`

6. **Роль*** - Выберите роль пользователя:
   - **Администратор** - полный доступ к системе
   - **МОЛ** - управление материалами и заявками
   - **Руководитель** - функции МОЛ + управление пользователями
   - **Инженер** - создание заявок, просмотр каталога

#### Опциональные поля

7. **Подразделение** - Отдел компании
   - Выбор из списка существующих подразделений

8. **Рабочая группа** - Для специалистов
   - АРМ - Автоматизированное рабочее место
   - ГОТО - Группа оперативно-технического обслуживания
   - ГОКС - Группа оперативно-контрольных систем

### Шаг 3: Сохранение

Нажмите кнопку **"Добавить"**

**Результат:**
- ✅ Пользователь создан
- ✅ Автоматически сохранен в базе данных
- ✅ Появится в таблице пользователей
- ✅ Может войти в систему немедленно

**Возможные ошибки:**
- ❌ "Логин уже существует" - выберите другой логин
- ❌ "Email уже используется" - укажите другой email
- ❌ "Заполните все обязательные поля" - проверьте форму

## ✏️ Редактирование пользователя

### Шаг 1: Открытие формы редактирования

Найдите нужного пользователя в таблице и нажмите кнопку **"✏️ Редактировать"**

### Шаг 2: Изменение данных

Откроется модальное окно с текущими данными пользователя:

#### Поля доступные для редактирования:

1. **ID пользователя** - только для просмотра (нельзя изменить)

2. **Логин*** - можно изменить
   - ⚠️ После изменения пользователь должен использовать новый логин

3. **Новый пароль** - опционально
   - **Оставьте пустым**, если не нужно менять пароль
   - **Введите новый пароль**, если нужно сбросить

4. **Email*** - можно изменить
   - Должен быть уникальным

5. **Имя*** и **Фамилия*** - можно изменить

6. **Роль*** - **ВАЖНО!** Можно изменить роль пользователя
   - Изменение роли меняет права доступа
   - Например: Инженер → Руководитель

7. **Подразделение** - можно изменить или удалить

8. **Рабочая группа** - можно изменить или удалить

9. **Активен** - чекбокс
   - ✓ Активен - пользователь может войти в систему
   - ✗ Неактивен - доступ запрещен

### Шаг 3: Сохранение изменений

Нажмите кнопку **"Сохранить"**

**Результат:**
- ✅ Данные обновлены в базе данных **автоматически**
- ✅ Изменения видны сразу в таблице
- ✅ Если изменен логин/пароль - пользователь должен войти заново

### Примеры изменений

#### Пример 1: Смена пароля
```
1. Откройте редактирование пользователя
2. В поле "Новый пароль" введите новый пароль
3. Нажмите "Сохранить"
4. Сообщите пользователю новый пароль
```

#### Пример 2: Повышение до руководителя
```
1. Откройте редактирование инженера
2. В поле "Роль" выберите "Руководитель"
3. Нажмите "Сохранить"
4. При следующем входе пользователь увидит панель руководителя
```

#### Пример 3: Деактивация пользователя
```
1. Откройте редактирование пользователя
2. Снимите галочку "Активен"
3. Нажмите "Сохранить"
4. Пользователь больше не сможет войти в систему
```

## 🗑️ Удаление пользователя

### Процесс удаления

1. Найдите пользователя в таблице
2. Нажмите кнопку **"🗑️ Удалить"**
3. Подтвердите удаление в диалоговом окне

**Предупреждение:**
```
Вы уверены, что хотите удалить пользователя "username"?

Пользователь будет деактивирован и не сможет войти в систему.
```

### Что происходит при удалении

❌ **Мягкое удаление (Soft Delete):**
- Пользователь **НЕ** удаляется физически из базы данных
- Устанавливается флаг `deleted = true` и `active = false`
- Пользователь **не может войти** в систему
- История его действий **сохраняется**
- Заявки созданные им **остаются** в системе

✅ **Преимущества:**
- Сохраняется история
- Можно восстановить пользователя
- Не нарушается целостность данных

### Неактивные пользователи в таблице

Удаленные (неактивные) пользователи:
- Показываются с пониженной прозрачностью (полупрозрачные)
- Статус: ✗ Неактивен
- Вместо кнопки "Удалить" показывается **"↻ Восстановить"**

---

## ↻ Восстановление пользователя

Если пользователь был удален по ошибке:

1. Найдите его в таблице (будет полупрозрачным)
2. Нажмите кнопку **"↻ Восстановить"**
3. Пользователь снова станет активным

**Результат:**
- ✅ Пользователь может войти в систему
- ✅ Все его данные сохранены
- ✅ История восстановлена

## 🆘 Решение проблем

### Проблема: Не могу добавить пользователя

**Причины:**
- Логин уже существует
- Email уже используется
- Не заполнены обязательные поля

**Решение:**
- Проверьте уникальность логина и email
- Заполните все поля со звездочкой (*)

### Проблема: Изменения не сохраняются

**Причины:**
- Проблемы с сетью
- Недостаточно прав
- Ошибка сервера

**Решение:**
- Проверьте подключение к интернету
- Обновите страницу (F5)
- Обратитесь к администратору

### Проблема: Пользователь не может войти

**Причины:**
- Неправильный логин или пароль
- Пользователь деактивирован
- Роль не назначена

**Решение:**
1. Проверьте статус пользователя в таблице
2. Убедитесь что "Активен" = ✓
3. Сбросьте пароль через редактирование
4. Проверьте что роль назначена
