# Руководство МОЛ и руководителя - Система учета ТМЦ

## Обзор

Материально-ответственное лицо (МОЛ) и руководитель имеют **практически идентичный функционал** для работы с каталогом ТМЦ, заявками и отчетностью. Руководитель дополнительно имеет доступ к управлению пользователями.

**Различия:**
- **МОЛ** - фокус на управлении материалами и обработке заявок
- **Руководитель** - все функции МОЛ + управление пользователями системы

---

## Вход в систему

### Для МОЛ:
1. Откройте браузер: `http://localhost:8080`
2. Введите логин и пароль МОЛ
3. Нажмите **"Войти"**
4. Автоматическое перенаправление на `/mol-dashboard.html`

### Для руководителя:
1. Откройте браузер: `http://localhost:8080`
2. Введите логин и пароль руководителя
3. Нажмите **"Войти"**
4. Автоматическое перенаправление на `/manager-dashboard.html`

**Тестовые учетные записи:**
- **МОЛ:** логин `mol_user`, пароль `mol123`
- **Руководитель:** логин `manager_user`, пароль `manager123`

---

### Главный экран МОЛ (`/mol-dashboard.html`)

После входа вы увидите экран с **7 карточками функций**:

4. **🛒 Заказ ТМЦ** - оформление заказов

**Каждая карточка:**
- Кликабельна - при клике открывается соответствующая страница
- Содержит иконку, название и краткое описание функции
- Подсвечивается при наведении

### Главный экран руководителя (`/manager-dashboard.html`)

Руководитель видит **те же функции МОЛ** плюс:

## Каталог ТМЦ

### Открытие каталога (`/mol-catalog.html`)

**Способ 1:** Нажмите карточку **"📦 Каталог ТМЦ"** на главном экране  
**Способ 2:** Выберите **"Каталог ТМЦ"** в левом меню

### Три склада

Система работает с **тремя типами складов**:

1. **Склад расходных материалов** (CONSUMABLES)
   - Расходные материалы одноразового использования
   - Примеры: краска, клей, растворители, смазка

2. **Склад запасных частей** (SPARE_PARTS)
   - Запасные части для оборудования
   - Примеры: подшипники, ремни, фильтры, клапаны

3. **Склад комплектующих** (COMPONENTS)
   - Электронные и механические компоненты
   - Примеры: резисторы, конденсаторы, микросхемы, болты

### Таблица каталога

Каталог отображается в виде таблицы со столбцами:

| Столбец | Описание |
|---------|----------|
| **Код** | Уникальный код материала (артикул) |
| **Название** | Полное наименование материала |
| **Категория** | Категория материала |
| **Склад** | На каком складе хранится |
| **Расположение** | Место хранения на складе (полка, стеллаж) |
| **Остаток** | Текущее количество на складе |
| **Единица** | Единица измерения (шт, кг, м и т.д.) |
| **Минимум** | Минимальный остаток для предупреждения |
| **Статус** | Цветовой индикатор уровня запасов |
| **Действия** | Кнопки редактирования и удаления |

### Индикаторы уровня запасов

**Цветовая индикация:**

- 🟢 **Зеленый** - В наличии (остаток >= минимального)
- 🟡 **Желтый** - Мало (остаток < минимального)
- 🔴 **Красный** - Отсутствует (остаток = 0)

### Добавление материала

**Шаги:**

1. Нажмите кнопку **"➕ Добавить материал"** (обычно вверху таблицы)
2. Откроется модальное окно с формой

**Поля формы:**

**Обязательные поля (*):**
- **Код материала*** - уникальный артикул (например: `RES-100K-01`)
- **Название*** - полное наименование (например: `Резистор 100кОм`)
- **Категория*** - выбор из списка категорий
- **Тип склада*** - CONSUMABLES / SPARE_PARTS / COMPONENTS
- **Единица измерения*** - шт, кг, м, л и т.д.

**Опциональные поля:**
- **Описание** - дополнительная информация о материале
- **Расположение** - где хранится на складе (стеллаж А-5, полка 3)
- **Текущее количество** - начальный остаток (по умолчанию 0)
- **Минимальный запас** - порог для предупреждения (например, 10)
- **Максимальный запас** - максимальный остаток
- **Критический запас** - критический уровень (обычно 5)
- **Цена за единицу** - стоимость материала
- **Производитель** - название производителя
- **Поставщик** - откуда поставляется
- **Штрих-код** - штрих-код материала
- **Условия хранения** - особые требования (температура, влажность)

3. Нажмите **"Добавить"** для сохранения
4. Материал появится в таблице каталога

**Важно:**
- Код материала должен быть уникальным
- Если код уже существует - появится ошибка
- Категория должна соответствовать типу склада

### Редактирование материала

**Шаги:**

1. Найдите материал в таблице
2. Нажмите кнопку **"✏️ Редактировать"** в столбце "Действия"
3. Откроется модальное окно с текущими данными материала
4. Измените нужные поля
5. Нажмите **"Сохранить"**

**Что можно редактировать:**
- Название, описание
- Категорию, расположение
- Остаток (текущее количество)
- Минимальный/максимальный/критический запас
- Цену, производителя, поставщика
- Условия хранения

**Ограничения:**
- Код материала **нельзя изменить** после создания (для сохранения целостности данных)
- При необходимости - создайте новый материал и удалите старый

### Удаление материала

**Шаги:**

1. Найдите материал в таблице
2. Нажмите кнопку **"🗑️ Удалить"**
3. Подтвердите удаление в диалоговом окне

**Что происходит:**
- Материал **не удаляется физически** из базы данных
- Выполняется **мягкое удаление (soft delete)**
- Устанавливается флаг `deleted = true` и `active = false`
- Материал исчезает из каталога, но данные сохраняются

**Важно:**
- Если материал используется в активных заявках - удаление может быть заблокировано
- Для восстановления удаленного материала обратитесь к администратору

### Фильтрация и поиск

**Фильтры:**

1. **По типу склада:**
   - Выберите из выпадающего списка: "Все склады", "Расходные материалы", "Запасные части", "Комплектующие"

2. **По категории:**
   - Выберите категорию из списка
   - Показываются только материалы выбранной категории

3. **По уровню запасов:**
   - "Все" - показать все
   - "В наличии" - остаток >= минимума
   - "Мало" - остаток < минимума
   - "Отсутствует" - остаток = 0

4. **По коду материала:**
   - Введите код в поле поиска
   - Поиск выполняется в реальном времени

5. **По названию:**
   - Введите часть названия
   - Система найдет все материалы, содержащие введенный текст

**Комбинированные фильтры:**
- Можно использовать несколько фильтров одновременно
- Например: "Запасные части" + "Мало" покажет только запасные части с низким остатком

**Кнопки управления фильтрами:**
- **"Применить"** - применить фильтры
- **"Сбросить"** - очистить все фильтры и показать все материалы

## Заявки на ТМЦ

### Открытие заявок (`/mol-requests.html`)

**Способ 1:** Нажмите карточку **"📝 Заявки на ТМЦ"** на главном экране  
**Способ 2:** Выберите **"Заявки на ТМЦ"** в левом меню

### Таблица заявок

Таблица отображает **все заявки от специалистов** со столбцами:

| Столбец | Описание |
|---------|----------|
| **☑️ Выбор** | Checkbox для массового выбора |
| **Номер заявки** | Уникальный номер (например: REQ-2026-001) |
| **Дата** | Дата создания заявки |
| **Статус** | Текущий статус (цветной индикатор) |
| **Специалист** | ФИО создателя заявки |
| **Группа** | Рабочая группа (АРМ / ГОТО / ГОКС) |
| **Работы** | Наименование работ (для чего материалы) |
| **Действия** | Кнопки обработки заявки |

### Статусы заявок

**Возможные статусы:**

- **Открыта** (🔵) - заявка создана, ожидает обработки
- **В работе** (🟡) - заявка принята МОЛ в работу
- **Ожидает выдачи** (🟠) - заявка одобрена, материалы готовы к выдаче
- **Выдана** (🟢) - материалы выданы специалисту
- **Закрыта** (✅) - заявка полностью выполнена
- **Отклонена** (🔴) - заявка отклонена с указанием причины

### Просмотр деталей заявки

**Как открыть:**

1. Найдите заявку в таблице
2. Нажмите на номер заявки или кнопку **"👁️ Просмотр"**
3. Откроется модальное окно с полной информацией

**Информация в окне:**

- **Номер заявки** и **Дата создания**
- **Заявитель** - ФИО и группа
- **Подразделение** - отдел заявителя
- **Назначение** - для чего нужны материалы
- **Примечания** - дополнительные комментарии
- **Таблица материалов:**
  - Код материала
  - Название
  - Запрошенное количество
  - Выданное количество (если выдан)
  - Единица измерения

### Принятие заявки в работу

**Процесс:**

1. Найдите заявку со статусом **"Открыта"**
2. Выберите заявку checkbox'ом (для массового выбора)
   - Или работайте с одной заявкой
3. Нажмите кнопку **"🟡 Взять в работу"** (обычно внизу таблицы)
4. Подтвердите действие

**Что происходит:**
- Статус заявки меняется на **"В работе"**
- **Автоматически отправляется push-уведомление** специалисту
- Уведомление: "Ваша заявка №XXX принята в работу"

**Массовые операции:**
- Можно выбрать несколько заявок checkbox'ами
- Нажать "Взять в работу" - все выбранные заявки будут приняты
- Каждому специалисту придет уведомление

### Выдача материалов

**Процесс:**

1. Заявка должна быть в статусе **"В работе"** или **"Ожидает выдачи"**
2. Подготовьте материалы согласно заявке
3. Нажмите кнопку **"✅ Выдать материалы"** (в деталях заявки или таблице)
4. В появившемся окне укажите **количество выданного** для каждого материала
   - Если выдано меньше запрошенного - укажите фактическое количество
   - Если выдано все - система подставит запрошенное количество
5. Нажмите **"Подтвердить выдачу"**

**Что происходит:**
- Статус заявки меняется на **"Выдана"**
- **Остатки материалов автоматически уменьшаются** в каталоге
- Создается запись о движении материала (тип: ISSUANCE)
- **Автоматически отправляется push-уведомление** специалисту
- Уведомление: "По заявке №XXX материалы выданы"

**Проверка остатков:**
- Если материала недостаточно - система покажет предупреждение
- Можно выдать меньше запрошенного количества
- Если материала нет на складе - нужно сначала пополнить

### Отклонение заявки

**Когда отклонять:**
- Материал отсутствует и не планируется поступление
- Неправильно указан материал
- Неоправданно большое количество
- Нарушение процедуры подачи заявки

**Процесс:**

1. Найдите заявку в таблице
2. Нажмите кнопку **"❌ Отклонить"** или **"Отменить заявку"**
3. В появившемся окне **обязательно укажите причину отклонения:**
   - Примеры: "Материал отсутствует на складе", "Неправильный код материала", "Превышен лимит выдачи"
4. Нажмите **"Подтвердить отклонение"**

**Что происходит:**
- Статус заявки меняется на **"Отклонена"**
- **Автоматически отправляется push-уведомление** специалисту
- Уведомление: "Ваша заявка №XXX отклонена. Причина: [указанная причина]"
- Специалист увидит причину отклонения в деталях заявки

**Важно:**
- Всегда указывайте причину отклонения
- Причина видна специалисту - это повышает прозрачность процесса

### Экспорт заявок

**Экспорт в PDF:**

1. Выберите заявки checkbox'ами (или одну заявку)
2. Нажмите кнопку **"📄 Экспорт в PDF"**
3. Файл автоматически скачается

**Содержание PDF:**
- Список выбранных заявок
- Детальная информация по каждой заявке
- Таблица материалов для каждой заявки
- Даты, статусы, заявители
- Форматирование для печати

**Экспорт в Excel:**

1. Выберите заявки checkbox'ами
2. Нажмите кнопку **"📊 Экспорт в Excel"**
3. Файл `.xlsx` автоматически скачается

**Содержание Excel:**
- Таблица с заявками (один лист)
- Столбцы: номер, дата, статус, заявитель, группа, работы, материалы
- Удобно для дальнейшей обработки и анализа

**Массовый экспорт:**
- Можно выбрать несколько заявок одновременно
- Экспортируется вся информация о выбранных заявках
- Удобно для подготовки отчетов

## История заявок

### Открытие истории (`/mol-history.html`)

**Способ 1:** Нажмите карточку **"📜 История заявок"** на главном экране  
**Способ 2:** Выберите **"История заявок"** в левом меню

### Таблица истории

**Автоматическое сохранение:**
- Все заявки **автоматически попадают в историю** при создании
- Не нужно вручную добавлять записи
- История содержит все когда-либо созданные заявки

**Столбцы таблицы:**

| Столбец | Описание |
|---------|----------|
| **☑️ Выбор** | Checkbox для массового экспорта |
| **Номер заявки** | Уникальный номер заявки |
| **Дата заявки** | Дата создания заявки |
| **Специалист** | ФИО создателя заявки |
| **Группа** | Рабочая группа (АРМ / ГОТО / ГОКС) |
| **Работы** | Наименование работ |
| **Статус** | Текущий статус заявки |
| **Дата выдачи** | Когда материалы были выданы (если выданы) |

### Фильтрация истории

**Фильтры:**

1. **По датам:**
   - **Дата с:** - начало периода
   - **Дата по:** - конец периода
   - Пример: показать заявки за последний месяц

2. **По рабочей группе:**
   - Выберите из списка: "Все группы", "АРМ", "ГОТО", "ГОКС"
   - Показываются заявки только выбранной группы

3. **По статусу:**
   - Выберите статус из списка
   - Показываются заявки только выбранного статуса

**Применение фильтров:**
- Нажмите кнопку **"Применить"** для фильтрации
- Нажмите **"Сбросить"** для очистки фильтров

### Экспорт истории

**Экспорт в PDF:**

1. При необходимости примените фильтры
2. Выберите записи checkbox'ами (или все отображаемые)
3. Нажмите **"📄 Экспорт в PDF"**
4. Файл скачается с выбранными записями

**Экспорт в Excel:**

1. Выберите записи checkbox'ами
2. Нажмите **"📊 Экспорт в Excel"**
3. Файл `.xlsx` скачается

**Примечание:**
- Если фильтры применены - экспортируются только отфильтрованные записи
- Если ничего не выбрано - экспортируются все видимые записи

## Списание ТМЦ

### Открытие списания

**Способ:** Нажмите карточку **"🗑️ Списание ТМЦ"** на главном экране

### Процесс списания

**Когда списывать:**
- Материалы испорчены или утрачены
- Истек срок годности
- Технологические потери
- Инвентаризация выявила недостачу

**Шаги:**

1. Выберите материал из каталога (поиск по коду или названию)
2. Укажите количество для списания
3. Укажите причину списания:
   - Испорчен
   - Утрачен
   - Истек срок годности
   - Технологические потери
   - Другое (с описанием)
4. Нажмите **"Списать"**

**Что происходит:**
- **Остаток материала автоматически уменьшается** в каталоге
- Создается запись о движении материала (тип: WRITE_OFF)
- Запись сохраняется в журнале движений материалов
- Если остаток стал ниже минимума - система покажет предупреждение

## Заказ ТМЦ

### Открытие заказа

**Способ:** Нажмите карточку **"🛒 Заказ ТМЦ"** на главном экране

### Процесс заказа

**Когда заказывать:**
- Остаток материала ниже минимального уровня
- Материал отсутствует на складе
- Плановое пополнение запасов

**Шаги:**

1. Выберите материал из каталога (или создайте новый материал)
2. Укажите количество для заказа
3. Укажите поставщика (если есть постоянный поставщик)
4. Укажите ожидаемую дату поставки
5. Добавьте примечания (если нужны особые требования)
6. Нажмите **"Создать заказ"**

**Что происходит:**
- Создается запись о заказе
- Заказ сохраняется в системе (для отслеживания)
- Можно экспортировать список заказов для отправки поставщику
- При поступлении материалов заказ автоматически связывается с поступлением

**Примечание:**  
Функционал заказа ТМЦ может быть интегрирован с ERP системой для автоматического создания заказов.

МОЛ и руководитель **не получают push-уведомления** о новых заявках в системе (в отличие от специалистов).

**Уведомления для МОЛ/руководителя:**

- Системные уведомления

**Как проверить новые заявки:**
- Регулярно открывайте раздел **"Заявки на ТМЦ"**
- Новые заявки имеют статус **"Открыта"** и отображаются вверху таблицы
- Рекомендуется проверять заявки несколько раз в день

## Типичные сценарии работы

### Сценарий 1: Поступление нового материала

1. Откройте **"Каталог ТМЦ"**
2. Выберите соответствующий склад (расходные/запасные/комплектующие)
3. Нажмите **"➕ Добавить материал"**
4. Заполните все поля (код, название, категория и т.д.)
5. Укажите текущее количество (сколько поступило)
6. Нажмите **"Добавить"**
7. Материал появится в каталоге с указанным остатком

### Сценарий 2: Обработка заявки от специалиста

1. Откройте **"Заявки на ТМЦ"**
2. Найдите новую заявку со статусом **"Открыта"**
3. Нажмите на номер заявки для просмотра деталей
4. Проверьте наличие материалов на складе:
   - Если все материалы в наличии - нажмите **"Взять в работу"**
   - Если материалов нет - отклоните заявку с указанием причины
5. Подготовьте материалы согласно заявке
6. Нажмите **"Выдать материалы"**
7. Укажите фактическое количество для каждого материала
8. Нажмите **"Подтвердить выдачу"**
9. Материалы выданы, заявка в статусе "Выдана"

### Сценарий 3: Списание испорченного материала

1. Откройте **"Списание ТМЦ"** (или **"Каталог ТМЦ"**)
2. Найдите материал (поиск по коду или названию)
3. Нажмите **"Списать"** или выберите действие "Списание"
4. Укажите количество для списания
5. Выберите причину: "Испорчен" или "Другое" с описанием
6. Нажмите **"Подтвердить списание"**
7. Остаток материала уменьшится в каталоге

### Сценарий 4: Подготовка отчета для руководства

1. Откройте **"Отчеты"**
2. Выберите тип отчета: "Отчет по расходу материалов"
3. Укажите период: последний месяц
4. При необходимости отфильтруйте по подразделению
5. Нажмите **"Сформировать отчет"**
6. Просмотрите отчет в таблице
7. Нажмите **"Экспорт в Excel"** или **"Экспорт в PDF"**
8. Файл скачается - отправьте его руководству

### Сценарий 5: Экспорт истории для архива

1. Откройте **"История заявок"**
2. Примените фильтр по датам: весь прошедший год
3. Выберите все записи (checkbox "Выбрать все")
4. Нажмите **"Экспорт в PDF"**
5. Файл скачается - сохраните для архива

## Часто задаваемые вопросы

**В: Можно ли изменить код материала после создания?**  
О: Нет, код материала нельзя изменить. Это нужно для сохранения целостности данных. Если нужен другой код - создайте новый материал.

**В: Что делать если материал отсутствует, а заявка уже принята в работу?**  
О: Отклоните заявку с указанием причины "Материал отсутствует на складе". Специалист получит уведомление.

**В: Можно ли восстановить удаленный материал?**  
О: Технически можно, но это требует доступа к базе данных. Обратитесь к администратору.

**В: Как часто нужно проверять заявки?**  
О: Рекомендуется проверять заявки 2-3 раза в день (утром, днем, вечером).

**В: Можно ли изменить количество выданного материала после выдачи?**  
О: Нет, после выдачи изменение невозможно. Если нужно дополнительное количество - создайте корректировку через списание/поступление.

**В: Как экспортировать данные для бухгалтерии?**  
О: Используйте функцию "Отчеты" → "Отчет по движениям" за нужный период → Экспорт в Excel.

**В: Можно ли работать с несколькими складами одновременно?**  
О: Да, можно переключаться между складами через фильтр в каталоге.

**В: Что делать если остаток материала показывает неправильное количество?**  
О: Используйте функцию "Корректировка остатка" (доступна в движении материалов) или обратитесь к администратору.
