## 🔧 Обновления Backend

### 1. ✅ UserController - расширение прав

**Файл:** `src/main/java/com/systemtmc/inventory/controller/UserController.java`

**Изменения:**

**До:**
```java
@PostMapping
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<UserDTO> createUser(...);

@PutMapping("/{id}")
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<UserDTO> updateUser(...);

@DeleteMapping("/{id}")
@PreAuthorize("hasRole('ADMIN')")
public ResponseEntity<Void> deleteUser(...);
```

**После:**
```java
@PostMapping
@PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')")  // ← MANAGER добавлен
public ResponseEntity<UserDTO> createUser(...);

@PutMapping("/{id}")
@PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')")  // ← MANAGER добавлен
public ResponseEntity<UserDTO> updateUser(...);

@DeleteMapping("/{id}")
@PreAuthorize("hasAnyRole('ADMIN', 'MANAGER')")  // ← MANAGER добавлен
public ResponseEntity<Void> deleteUser(...);
```

**Цель:** Предоставить руководителям права на управление пользователями

## ⚠️ Важные примечания

### Безопасность:

1. **Soft Delete**
   - Пользователи НЕ удаляются физически
   - Устанавливается флаг `deleted = true`
   - История сохраняется

2. **Изменение пароля**
   - Пустое поле = пароль не меняется
   - Заполненное поле = новый пароль
   - Backend логика обрабатывает это автоматически

3. **Права доступа**
   - MANAGER может управлять всеми пользователями
   - MANAGER не может изменять системные настройки (только ADMIN)

### Лучшие практики:

1. **Деактивируйте, не удаляйте**
   - Снимите галочку "Активен"
   - Или используйте кнопку "Удалить"
   - Это сохранит историю

2. **Проверяйте роли**
   - Перед повышением до MANAGER
   - Убедитесь что пользователь понимает ответственность

3. **Используйте сложные пароли**
   - При создании новых пользователей
   - Минимум 8 символов
