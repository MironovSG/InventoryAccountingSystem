# Система учета и выдачи товарно-материальных ценностей (ТМЦ)

Веб-приложение для автоматизации процессов учета, хранения, выдачи и контроля расхода товарно-материальных ценностей в сервисном центре производственного предприятия.

## Технологический стек

- **Backend**: Java 17, Spring Boot 3.2.1
- **Database**: PostgreSQL
- **Security**: Spring Security, JWT
- **Build Tool**: Maven
- **Documentation**: Swagger/OpenAPI
- **Monitoring**: Spring Actuator, Prometheus

## Основные функции

### 1. Подсистема учета материалов
- Ведение электронного учета ТМЦ
- Фильтрация по категориям, подразделениям и наличию на складе
- Цветовая маркировка остатков (зеленый – в наличии, желтый – мало, красный – нет)
- История изменений по каждой единице ТМЦ

### 2. Подсистема управления заявками
- Подача и обработка заявок на выдачу ТМЦ
- Отслеживание жизненного цикла заявок
- Автоматическая регистрация и история заявок
- Уведомления о статусах заявок

### 3. Подсистема отчетности и аналитики
- Формирование стандартных отчетов
- Статистика заявок и расхода ТМЦ
- Экспорт отчетов в Excel и PDF
- Автоматическая рассылка отчетов по расписанию

### 4. Подсистема безопасности и доступа
- Авторизация по логину и паролю (JWT)
- Разграничение прав доступа по ролям
- Журнал аудита действий пользователей

### 5. Подсистема интеграции
- REST API для интеграции с внешними системами
- Поддержка интеграции с ERP, Active Directory

## Роли пользователей

- **ADMIN** - Администратор системы
  - Управление пользователями и настройками
  - Полный доступ ко всем функциям
  - Панель: `/dashboard.html`

- **MOL** - Материально-ответственное лицо
  - Управление каталогом ТМЦ (3 склада)
  - Обработка заявок от специалистов
  - Списание и заказ ТМЦ
  - Формирование отчетов
  - Панель: `/mol-dashboard.html`

- **ENGINEER** - Инженер / Специалист
  - Просмотр каталога ТМЦ
  - Создание заявок на выдачу
  - Отслеживание своих заявок
  - Просмотр истории всех заявок
  - Получение push-уведомлений
  - Панель: `/user-dashboard.html`

- **MANAGER** - Руководитель подразделения
  - Все функции ENGINEER
  - Просмотр заявок подразделения
  - Расширенная аналитика
  - Панель: `/user-dashboard.html`

## Установка и настройка

### Предварительные требования

- JDK 17 или выше
- Maven 3.6+
- PostgreSQL 13+

### Шаг 1: Клонирование репозитория

```bash
git clone <repository-url>
cd Inventory-accounting-and-issuance-system
```

### Шаг 2: Настройка базы данных

Создайте базу данных PostgreSQL:

```sql
CREATE DATABASE inventory_db;
CREATE USER inventory_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE inventory_db TO inventory_user;
```

### Шаг 3: Настройка конфигурации

Отредактируйте файл `src/main/resources/application.yml`:

```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/inventory_db
    username: inventory_user
    password: your_password
```

### Шаг 4: Сборка проекта

```bash
mvn clean install
```

### Шаг 5: Запуск приложения

```bash
mvn spring-boot:run
```

Или запустите собранный JAR:

```bash
java -jar target/inventory-system-1.0.0.jar
```

## API Endpoints

### Аутентификация
- `POST /auth/login` - Вход в систему
- `GET /auth/me` - Получить текущего пользователя

### Пользователи
- `GET /users` - Получить всех пользователей
- `GET /users/{id}` - Получить пользователя по ID
- `POST /users` - Создать пользователя
- `PUT /users/{id}` - Обновить пользователя
- `DELETE /users/{id}` - Удалить пользователя

### Материалы
- `GET /materials` - Получить все материалы
- `GET /materials/{id}` - Получить материал по ID
- `GET /materials/low-stock` - Получить материалы с низким запасом
- `POST /materials` - Создать материал
- `PUT /materials/{id}` - Обновить материал
- `DELETE /materials/{id}` - Удалить материал

### Заявки
- `GET /requests` - Получить все заявки
- `GET /requests/{id}` - Получить заявку по ID
- `GET /requests/my-requests` - Получить мои заявки ⭐ НОВОЕ
- `POST /requests` - Создать заявку (с push-уведомлением МОЛ) ⭐ ОБНОВЛЕНО
- `POST /requests/{id}/approve` - Одобрить заявку
- `POST /requests/{id}/issue` - Выдать материалы
- `POST /requests/{id}/reject` - Отклонить заявку

### Уведомления ⭐ НОВОЕ
- `GET /notifications` - Получить все уведомления
- `GET /notifications/unread` - Получить непрочитанные
- `GET /notifications/unread/count` - Количество непрочитанных
- `PUT /notifications/{id}/read` - Отметить как прочитанное
- `PUT /notifications/read-all` - Отметить все как прочитанные

### Движения материалов
- `GET /movements/material/{id}` - История движений материала
- `POST /movements/receipt` - Поступление материала
- `POST /movements/adjust` - Корректировка остатка

## Новые функции (v2.0) ⭐

### Функционал для специалистов (пользователей)

**Страницы:**
- `/user-dashboard.html` - Главная панель пользователя
- `/user-catalog.html` - Просмотр каталога ТМЦ (только чтение)
- `/user-my-requests.html` - Мои заявки
- `/user-history.html` - История всех заявок
- `/user-notifications.html` - Уведомления

**Возможности:**
- 📦 Просмотр каталога материалов с фильтрами
- ➕ Создание заявок на выдачу ТМЦ через модальное окно
- 📝 Отслеживание статуса своих заявок
- 📜 Просмотр истории всех заявок в системе
- 🔔 Push-уведомления в реальном времени
- 🎯 Простой и интуитивный интерфейс

**Push-уведомления:**
- При создании заявки → МОЛ получает уведомление
- При принятии заявки в работу → Пользователь получает уведомление
- При выдаче материалов → Пользователь получает уведомление
- При отклонении заявки → Пользователь получает уведомление с причиной

- `/mol-catalog.html` - Управление каталогом ТМЦ
