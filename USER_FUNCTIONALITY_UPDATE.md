## Схема работы с заявками (для пользователей)

```
1. ПОЛЬЗОВАТЕЛЬ заходит в систему
   └─> Перенаправление на /user-dashboard.html
   
2. ПОЛЬЗОВАТЕЛЬ просматривает каталог
   └─> /user-catalog.html
   └─> Применяет фильтры
   └─> Находит нужные материалы
   └─> Запоминает коды материалов
   
3. ПОЛЬЗОВАТЕЛЬ создает заявку
   └─> Нажимает "Создать заявку"
   └─> Открывается модальное окно
   └─> Заполняет таблицу:
       - Код материала (обязательно)
       - Название (обязательно)
       - Количество (обязательно)
       - Наименование работ (обязательно)
   └─> Может добавить дополнительные строки
   └─> Нажимает "Отправить в работу"
   
4. СИСТЕМА обрабатывает заявку
   └─> Проверяет существование материалов
   └─> Создает заявку со статусом ACCEPTED
   └─> Отправляет push-уведомление ВСЕМ МОЛ
   └─> Показывает подтверждение пользователю
   
5. МОЛ получает уведомление
   └─> На его главном экране появляется:
       "Поступила новая заявка №REQ-... от [ФИО] ([Подразделение])"
   └─> Красный бадж с количеством уведомлений
   
6. МОЛ обрабатывает заявку
   └─> Принимает в работу
   └─> ПОЛЬЗОВАТЕЛЬ получает push: "Заявка принята в работу"
   
7. ПОЛЬЗОВАТЕЛЬ отслеживает статус
   └─> /user-my-requests.html
   └─> Видит статус "В работе" (🟡)
   └─> Может просмотреть детали
   
8. МОЛ выдает материалы
   └─> Статус меняется на "Выдана"
   └─> ПОЛЬЗОВАТЕЛЬ получает push: "Материалы выданы"
   
9. Заявка автоматически попадает в историю
   └─> /user-history.html
   └─> Доступна для просмотра всем пользователям
```

## Push-уведомления в реальном времени

### Механизм работы:

1. **Polling каждые 30 секунд**
   - Автоматический запрос `/notifications/unread`
   - Обновление счетчика и списка уведомлений

2. **Визуализация:**
   - Красный бадж с количеством непрочитанных
   - Последние 5 уведомлений на главной странице
   - Клик по бадж → `/user-notifications.html`

3. **Триггеры для пользователей:**
   - МОЛ принял заявку в работу → push пользователю
   - Материалы выданы → push пользователю
   - Заявка отклонена → push пользователю с причиной

4. **Триггеры для МОЛ:**
   - Новая заявка создана → push всем МОЛ

## Валидация на фронтенде

### При создании заявки:

✅ **Проверяется:**
- Заполнена хотя бы одна строка полностью
- Код материала существует в каталоге
- Количество > 0
- Наименование работ указано

❌ **Ошибки отображаются:**
- "Заполните хотя бы одну строку с материалом"
- "Материал с кодом 'XXX' не найден в каталоге"
- "Ошибка при создании заявки: [детали]"

## API Endpoints (новые/обновленные)

### Для заявок:
```
GET  /api/requests/my-requests    ← Новый! Мои заявки
POST /api/requests                 ← Обновлен (отправка push МОЛ)
```

### Для материалов:
```
GET  /api/materials/search?query=...  ← Используется при создании заявки
```

### Для уведомлений:
```
GET  /api/notifications               ← Все уведомления
GET  /api/notifications/unread        ← Непрочитанные
PUT  /api/notifications/{id}/read     ← Отметить прочитанным
PUT  /api/notifications/read-all      ← Отметить все
```
