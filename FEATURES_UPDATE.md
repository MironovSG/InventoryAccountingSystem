# Обновления функционала системы учета ТМЦ

## Новые возможности для МОЛ (Материально-ответственное лицо)

### 1. Главный экран МОЛ (`/mol-dashboard.html`)

Специализированный интерфейс с карточками функций:

#### Доступные разделы:
- ** Каталог ТМЦ** - управление материалами на всех складах
- ** Заявки на ТМЦ** - обработка заявок от специалистов
- ** Списание ТМЦ** - списание материалов
- ** Заказ ТМЦ** - оформление заказов материалов
- ** История заявок** - полная история всех операций
- ** Отчеты** - аналитические отчеты
- ** Экспорт в ERP** - интеграция с ERP системой

### 2. Каталог ТМЦ (`/mol-catalog.html`)

#### Три склада:
- **Склад расходных материалов** (CONSUMABLES)
- **Склад запасных частей** (SPARE_PARTS)
- **Склад комплектующих** (COMPONENTS)

#### Функции:
-  Добавление новых материалов с указанием:
  - Код материала
  - Название
  - Категория (в зависимости от склада)
  - Расположение на складе
  - Остаток
  - Минимальный запас
  
-  Редактирование существующих материалов
-  Удаление материалов
-  Фильтрация по:
  - Типу склада
  - Категории
  - Уровню запасов (в наличии / мало / отсутствует)
  
-  Поиск по коду или названию

**Только МОЛ имеет права на редактирование перечня материалов!**

### 3. Заявки на ТМЦ (`/mol-requests.html`)

#### Табличный вид с колонками:
- **Выбор** - checkbox для массового выбора
- **Номер заявки** - уникальный номер
- **Дата заявки** - дата создания
- **Статус** - текущий статус заявки
- **Специалист** - кто создал заявку
- **Группа** - рабочая группа (АРМ / ГОТО / ГОКС)
- **Наименование работ** - для чего нужны материалы

#### Функции:
-  **Массовый выбор заявок** - checkbox для выбора нескольких заявок
-  **Экспорт в PDF** - выгрузка выбранных заявок в PDF
-  **Экспорт в Excel** - выгрузка выбранных заявок в Excel
-  **Взять в работу** - принять заявки в обработку
-  **Отменить заявку** - отклонить заявки с указанием причины

#### Push-уведомления:
При изменении статуса заявки **пользователю автоматически приходит push-уведомление** в веб-приложение:
-  "Заявка принята в работу"
-  "Материалы выданы"
-  "Заявка отклонена" (с указанием причины)

### 4. История заявок (`/mol-history.html`)

#### Большая таблица со всеми заявками:
- ** Выбор** - для массового экспорта
- **Номер заявки**
- **Дата заявки**
- **Специалист** - создатель заявки
- **Группа** - АРМ / ГОТО / ГОКС
- **Наименование работ**
- **Статус**
- **Дата выдачи**

#### Функции:
-  **Автоматическое сохранение** - все заявки автоматически попадают в историю
-  **Фильтрация** по:
  - Дате (с ... по ...)
  - Рабочей группе (АРМ / ГОТО / ГОКС)
  
-  **Массовый выбор записей**
-  **Экспорт в PDF** - выбранных записей
-  **Экспорт в Excel** - выбранных записей

## Новые Backend компоненты

### 1. Модели данных

#### WarehouseType (enum)
```java
CONSUMABLES("Склад расходных материалов")
SPARE_PARTS("Склад запасных частей")
COMPONENTS("Склад комплектующих")
```

#### WorkGroup (enum)
```java
ARM("АРМ")
GOTO("ГОТО")
GOKS("ГОКС")
```

#### Notification (entity)
Новая таблица для хранения push-уведомлений:
- ID пользователя
- Заголовок
- Сообщение
- Тип уведомления
- Прочитано/не прочитано
- Связанная сущность (заявка, материал и т.д.)

### 2. Новые API endpoints

#### Уведомления
```
GET  /api/notifications               - получить все уведомления
GET  /api/notifications/unread        - непрочитанные уведомления
GET  /api/notifications/unread/count  - количество непрочитанных
PUT  /api/notifications/{id}/read     - отметить как прочитанное
PUT  /api/notifications/read-all      - отметить все как прочитанные
```

#### Экспорт отчетов
```
GET /api/reports/requests/export/pdf?startDate=...&endDate=...
GET /api/reports/requests/export/excel?startDate=...&endDate=...
```

### 3. Сервисы

#### PushNotificationService
- `sendRequestAcceptedNotification()` - уведомление о принятии заявки
- `sendRequestIssuedNotification()` - уведомление о выдаче
- `sendRequestRejectedNotification()` - уведомление об отклонении
- `getUserNotifications()` - получить все уведомления пользователя
- `getUnreadNotifications()` - непрочитанные уведомления
- `markAsRead()` - отметить как прочитанное

#### ReportService
- `exportRequestsReportToExcel()` - экспорт отчета по заявкам в Excel
- `exportMaterialConsumptionReportToExcel()` - экспорт по расходу материалов
- Формирование статистики и аналитики

## Рабочие группы специалистов

Добавлена поддержка трех рабочих групп:

### АРМ (Автоматизированное рабочее место)
### ГОТО (Группа оперативно-технического обслуживания)
### ГОКС (Группа оперативно-контрольных систем)

Каждый специалист принадлежит к одной из этих групп, что отображается:
- В заявках
- В истории заявок
- В отчетах

## Интеграция компонентов

### Схема работы с заявками:

1. **Специалист создает заявку** 
   ↓
2. **Заявка автоматически появляется в разделе "Заявки на ТМЦ" у МОЛ**
   ↓
3. **МОЛ выбирает заявки и нажимает "Взять в работу"**
   ↓
4. **Система отправляет push-уведомление специалисту**: "Заявка №XXX принята в работу"
   ↓
5. **МОЛ обрабатывает заявку и выдает материалы**
   ↓
6. **Система отправляет уведомление**: "По заявке №XXX материалы выданы"
   ↓
7. **Заявка автоматически сохраняется в "Истории заявок"**

### Схема работы с каталогом:

1. **МОЛ открывает "Каталог ТМЦ"**
   ↓
2. **Выбирает нужный склад и категорию**
   ↓
3. **Добавляет/редактирует материалы** с указанием:
   - Кода
   - Названия
   - Расположения
   - Остатка
   ↓
4. **Система сохраняет изменения и обновляет каталог**

## Экспорт данных

### Форматы экспорта:
- ** PDF** - для печати и архивирования
- ** Excel** - для дальнейшей обработки и анализа

### Что можно экспортировать:
-  Заявки (выбранные)
-  История заявок (выбранные записи)
-  Отчеты по расходу материалов
-  Отчеты по заявкам за период

## Безопасность и права доступа

### Права МОЛ:
-  Полный доступ к каталогу ТМЦ (добавление, редактирование, удаление)
-  Просмотр и обработка всех заявок
-  Доступ к истории заявок
-  Формирование отчетов
-  Экспорт данных

### Права специалистов (ENGINEER, SPECIALIST_*):
-  Создание заявок на выдачу ТМЦ
-  Просмотр своих заявок
-  Получение push-уведомлений
-  НЕТ доступа к редактированию каталога

## База данных

### Новые поля:
- `materials.warehouse_type` - тип склада
- `users.work_group` - рабочая группа специалиста

### Новая таблица:
- `notifications` - хранение push-уведомлений

### SQL миграция:
Создан файл `V3__add_new_features.sql` для автоматического обновления БД при запуске.

## Технические детали

### Frontend:
- `mol-dashboard.html` - главный экран МОЛ с карточками функций
- `mol-catalog.html` - каталог ТМЦ с фильтрами
- `mol-requests.html` - обработка заявок с массовым выбором
- `mol-history.html` - история заявок с экспортом
- `mol-catalog.js` - логика работы с каталогом

### Backend:
- `PushNotificationService` - сервис уведомлений
- `NotificationRepository` - репозиторий уведомлений
- `NotificationController` - API для уведомлений
- Обновлен `RequestService` для отправки уведомлений

### Интеграция:
- Push-уведомления отправляются автоматически при изменении статуса заявки
- Все заявки автоматически сохраняются в истории
- Экспорт работает через REST API с передачей JWT токена

## Как использовать

### Для МОЛ:

1. Войти в систему под учетной записью МОЛ
2. Откроется главный экран с выбором функций
3. Выбрать нужную функцию (каталог, заявки, история и т.д.)
4. Работать с данными:
   - Добавлять/редактировать материалы в каталоге
   - Обрабатывать заявки (принимать, отклонять)
   - Просматривать историю
   - Формировать отчеты

### Для специалистов:

1. Войти в систему
2. Создать заявку на выдачу ТМЦ
3. Получить push-уведомление при изменении статуса
4. Просматривать статус своих заявок
